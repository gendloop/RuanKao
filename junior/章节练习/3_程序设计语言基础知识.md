2. 对于中缀形式的算术表达式a+b*c-d, 其后缀式为()
    * `a b c d + * -`
    * `a b + c * d -`
    * `a b c * + d -`
    * `a b c + d * -`

    C

    * 常见的算术表达式为运算符在两操作数中间, 即中缀表达式
    * 后缀表达式为栈的运用, 没有优先级判断, 没有括号处理, 便于计算机的计算
    * 中缀改后缀的方法为: 按运算符优先级加括号, 把运算符移到括号后面, 再去掉括号
    * 运算演示: `(A+B)*C => ((A+B)*C) => ((AB+)*C) => ((AB+)C*) => AB+C*`
    * 常见中缀,后缀对比

      |中缀|后缀|
      |:--|:--|
      | `A + B` | `A B +` |
      | `A + B * C` | `A B C * +` |
      | `(A + B) * C` | `A B + C *` |
      | `A * (B + C)` | `A B C + *` |
      | `A + B * C - D / E` | `A B C * + D E / -` |

3. 针对C语言源程序进行编译的过程, 下面说法中正确的是()
    * 需对未定义的变量报告错误
    * 需判断变量的值是否正确
    * 需计算循环语句的执行次数
    * 需判断循环条件是否正确

    A

    * 未定义的变量在编译时报告, 编译时处理静态语义
    * 变量的值, 循环条件的值及执行次数等属于动态语义, 在动行时处理
    * 对C语言源程序进行编译的过程包括
      * 预处理: 进行头文件展开, 宏替换, 条件编译, 删除注释, 生成.i中间文件
      * 编译: 对预处理后的.i文件, 进行词法分析, 语法分析, 语义分析和优化, 生成.s汇编文件
      * 汇编: 将.s汇编文件翻译成二进行的机器指令, 生成.o或.obj目标文件
      * 链接: 进行代码段合并和地址重定位, 生成最终可执行文件.exe
